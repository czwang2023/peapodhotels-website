附录 A: 智能定价详解

本附录包含智能定价功能的详细收入示例、折扣策略和业务影响分析。

基于时长的折扣（周折扣和月折扣）

自动降低长住客人的每晚价格。客人可以看到折后总价，鼓励延长预订。

折扣类型： 1. 周折扣（7-27 晚） - 基础价格的折扣百分比 - 范围：5% - 50% - 推荐：10-20% - 示例：$100/晚 → 7晚以上$90/晚 = 一周$630（原价$700）

  1. 月折扣（28晚以上）
    • 基础价格的折扣百分比
    • 范围：10% - 60%
    • 推荐：20-30%
    • 示例：$100/晚 → 28晚以上$75/晚 = 一月$2,100（原价$2,800）

即时优惠深度解析

当日预订（“今晚”）的特别折扣，用于填补原本收入为$0的空房。仅对高等级客人（白金/钻石）可见，作为忠诚度福利。

主要特点： - 仅限今晚：预订必须是当天 - 专属权益：仅白金/钻石等级客人可见 - 房东控制：您设置折扣百分比 - 自动过期：午夜过期（优惠结束，房间恢复正常价格） - 可选功能：可按房源开启/关闭

资格要求： - 客人等级：白金（10次以上预订）或钻石（25次以上预订） - 预订时间：当日（今天入住） - 房源可用性：今晚必须可用 - 房东设置：该房源已启用即时优惠

折扣叠加规则（重要！）

黄金法则： 仅适用单个最优折扣。折扣不叠加。

为什么不叠加？

防止过度折扣损害收入。如果没有此规则： - 20%月折扣 + 30%即时优惠 = 50%折扣（太多了！） - 房东亏损比空置更严重

示例场景：

场景 A：30晚住宿 + 即时优惠已启用 - 基础价格：$100/晚 - 月折扣：25%折扣 → $75/晚 - 即时优惠：40%折扣 → $60/晚（但不符合条件 - 预订是30晚，不是今晚） - 适用： 月折扣（$75/晚）✅

场景 B：7晚住宿 + 即时优惠已启用（今晚入住） - 基础价格：$100/晚 - 周折扣：15%折扣 → $85/晚 - 即时优惠：30%折扣 → $70/晚（符合条件！） - 适用： 即时优惠（$70/晚）（优于15%周折扣）

场景 C：10晚住宿（无即时优惠） - 基础价格：$100/晚 - 周折扣：15%折扣 → $85/晚 - 月折扣：25%折扣 → （不符合条件，少于28晚） - 适用： 周折扣（$85/晚）✅

系统如何决定：

系统计算预订的所有符合条件的折扣，并自动应用最优折扣（最低价格）。工作原理：

  1. 从基础价格开始（无折扣）
  2. 检查周折扣是否适用（7晚以上且功能已启用）
  3. 检查月折扣是否适用（28晚以上且功能已启用）
  4. 检查即时优惠是否适用（今晚入住、白金/钻石客人、功能已启用）
  5. 比较所有符合条件的价格，为客人选择最低价格

收入影响示例

示例 1：海滨别墅 - 智能定价前 - 基础价格：$200/晚 - 平均预订：3晚 - 入住率：60%（18天/月） - 收入：18 × $200 = $3,600/月

示例 1：海滨别墅 - 智能定价后 - 基础价格：$200/晚 - 周折扣：15% → 7晚以上$170/晚 - 月折扣：25% → 28晚以上$150/晚 - 预订变化： - 10晚短住（每次3晚）：10 × $200 = $2,000 - 2次周住（每次7晚）：14 × $170 = $2,380 - 1次即时优惠今晚：1 × $140（30%折扣）= $140 - 入住率：83%（25天/月）← 因折扣而增加 - 收入：$2,000 + $2,380 + $140 = $4,520/月 +26%收入！

示例 2：山间小屋 - 季节性策略 - 基础价格：$150/晚 - 冬季（旺季）：不启用折扣 - 夏季（淡季）：20%周折扣、30%月折扣 - 结果：夏季入住率从40%增加到65% - 年收入增长：+18%

示例 3：城市公寓 - 即时优惠 - 基础价格：$120/晚 - 即时优惠：40%折扣 → $72/晚 - 场景：本月5个空置夜晚原本收入$0 - 使用优惠后：5晚中3晚被白金客人预订 - 收入增益：3 × $72 = $216（vs $0）✅

示例 4：豪华别墅 - 策略性折扣 - 基础价格：$500/晚 - 周折扣：10% → $450/晚 - 月折扣：20% → $400/晚 - 目标：长期远程工作者和搬迁家庭 - 结果：平均预订时长从3晚增加到12晚 - 年入住率从55%增加到75% - 总收入增长：+35%

分析与绩效跟踪

联合房东关键指标： - 平均响应时间（目标：<1小时） - 每周消息量 - 客人满意度评价提及 - 预订转化率（咨询 → 确认） - 问题解决时间 - 主动客人互动

清洁人员关键指标： - 任务完成率（目标：100%） - 平均完成时间 - 照片上传合规性 - 问题报告提交 - 客人清洁度评分 - 重新清洁请求（目标：0%）

如何查看分析： 1. 访问房东仪表板 → 分析 2. 按团队成员筛选 3. 查看趋势（周、月、季度） 4. 与基准比较 5. 在定期检查中讨论 6. 根据需要调整角色/职责

税务影响： - 联合房东可能被视为承包商（美国1099表格） - 清洁人员通常是税务目的的承包商 - 咨询税务专业人员以正确分类 - 保留所有团队成员付款记录

保险： - 验证房屋保险是否涵盖短期租赁 - 考虑联合托管的额外责任保险 - 清洁人员应有自己的责任保险 - 如有要求，在保险披露中包含团队成员

合同与协议： - 创建书面联合托管协议 - 定义职责、付款条款、终止条款 - 包含客人信息保密协议 - 处理财产损坏和责任 - 指定知识产权（照片、描述）

付款结构： - 联合房东：收入百分比（10-20%）或每次预订固定费用 - 清洁人员：每次清洁固定费用（根据房产大小而异） - 考虑高评分的绩效奖金 - 每周或每月付款计划 - 尽可能使用平台付款以便记录

附录 E: 智能分配算法

智能分配如何工作

第1步：资格检查 - 筛选分配给此房产的清洁人员 - 检查清洁人员在截止时间的可用性 - 排除标记为不可用或超负荷的清洁人员

第2步：评分算法

每个符合条件的清洁人员根据以下因素获得评分：

  1. 当前工作量（40%权重）
    • 待处理任务的预估小时数总和
    • 工作量越低 = 分数越高
    • 公式：100 - (pending_hours / max_hours_per_week * 100)
  2. 绩效评分（30%权重）
    • 过去完成任务的平均评分
    • 评分：1-5星 → 20-100分
    • 公式：rating * 20
  3. 截止日期可行性（20%权重）
    • 能否在截止日期前完成？
    • 公式：(deadline_hours - pending_hours) / task_duration * 100
    • 如果结果 < 0（超负荷），该清洁人员分数 = 0
  4. 到房产的距离（10%权重）
    • 从清洁人员位置的行程时间（如果可用）
    • 尚未实现 - 保留用于未来增强
    • 当前：所有人平等分数（50分）

第3步：选择 - 总分最高的清洁人员被分配 - 平局：任务分配最少的清洁人员（终身） - 最终平局：按姓名字母顺序

示例场景

任务： 周五下午2点前需要手动深度清洁（预估3小时）

符合条件的清洁人员：

清洁人员 待处理小时 评分 截止小时 工作量分数 评分分数 可行性分数 总分
Sarah 6小时 4.8★ 48小时 85 (40×0.4=34) 96 (30×0.3=29) 100 (20×0.2=20) 83分
Mike 12小时 4.5★ 48小时 70 (40×0.4=28) 90 (30×0.3=27) 75 (20×0.2=15) 70分
Lisa 3小时 4.2★ 48小时 93 (40×0.4=37) 84 (30×0.3=25) 100 (20×0.2=20) 82分

结果： Sarah被分配（最高分）

备用逻辑

如果没有清洁人员在截止日期前可用： 1. 系统在所有清洁人员中找到最早可用时段 2. 向房东建议替代截止日期 3. 房东可以： - 接受新截止日期 - 手动分配并接受风险 - 取消任务创建

紧急通知系统

警报风险级别：

  • 🔴 紧急 - 需要立即采取行动
    • 任务未分配清洁人员
    • 距离客人入住不到3小时，任务未开始
  • 🟠 高 - 需要尽快采取行动
    • 距离入住不到6小时，任务未开始
  • 🟡 中 - 注意，监控情况
    • 距离入住不到12小时，任务未开始
  • 🟢 低 - 正常监控
    • 所有其他待处理任务

何时收到警报：

您将在手机上收到推送通知： - 立即：客人退房但没有清洁人员可在下次入住前分配（即时发送） - 立即：您创建手动任务但没有清洁人员在截止日期前可用 - 任务未分配清洁人员 - 入住即将到来但清洁尚未开始 - 系统检测到可能错过客人到达截止日期的风险

警报频率： - 紧急故障：无法分配退房清洁时立即发送（< 1分钟） - 常规检查：每30分钟自动运行 - 同一任务在1小时内不会收到重复警报 - 紧急警报具有最高优先级（覆盖勿扰模式）

立即紧急 - 退房无可用清洁人员：

🚨 紧急：无可用清洁人员
房产"Beachfront Coronado Home"没有可用清洁人员
进行退房清洁。客人将在6小时后入住。
请立即分配清洁人员！
[点击分配清洁人员]

响应警报： 1. 点击通知打开任务详情 2. 查看清洁人员状态 - 他们在处理吗？ 3. 采取行动： - 如果未开始，直接致电/短信清洁人员 - 重新分配给另一个可用清洁人员 - 如有必要延迟客人入住（最后手段） - 实时监控进度

防止误报：

系统对警报很智能： - 仅对真正有风险的任务发出警报 - 考虑清洁人员工作量和容量 - 在截止日期计算中考虑任务持续时间 - 清洁人员开始任务后停止警报 - 不对有充足时间的任务发出警报

附录 F: 故障排除

问：清洁人员拒绝了新时间表，怎么办？ 答：系统自动找下一个可用清洁人员。如果没有可用清洁人员，您会看到”需要手动干预”，可以延迟入住或直接联系清洁人员。

问：我可以覆盖自动重排决定吗？ 答：可以。在预检面板中，点击”手动模式”选择您自己的清洁人员或时间。

问：如果客人请求的变更超出我的政策怎么办？ 答：自动重排遵守您的取消政策和最短住宿设置。违反政策的请求需要手动批准。

问：如何为特定房源禁用自动重排？ 答：房源编辑器 → 设置 → 关闭”自动重排”（您将手动处理修改）。

附录 G: 最佳实践

任务管理最佳实践

创建任务： - 🎯 设置现实的截止日期 - 在下次入住前留出缓冲时间 - 🎯 提供清晰的备注 - 具体任务，不是模糊的指示 - 🎯 使用正确的优先级 - 将”紧急”保留给真正的紧急情况 - 🎯 准确估计持续时间 - 防止清洁人员超负荷 - 🎯 信任自动分配 - 算法考虑的因素比您更多

管理清洁人员： - 🎯 及时评分 - 在完成后24小时内提供反馈 - 🎯 公平评分 - 优秀工作5星，不是完美 - 🎯 直接沟通问题 - 使用任务备注或私信 - 🎯 监控工作量 - 确保团队成员之间均匀分配 - 🎯 认可表现优秀者 - 奖金、感谢信、持续分配

质量控制： - 🎯 审核照片 - 检查每个房间的前后照片 - 🎯 偶尔抽查 - 不通知访问房产（提前通知） - 🎯 客人反馈 - 跟踪评价中的清洁度评分 - 🎯 立即解决问题 - 不要让质量下滑 - 🎯 更新清单 - 为反复出现的问题添加项目

故障排除： - 🎯 完成延迟？ - 检查清洁人员工作量，调整估计 - 🎯 质量下降？ - 审核照片，提供具体反馈 - 🎯 清洁人员拒绝任务？ - 讨论可用性，调整时间表 - 🎯 客人投诉？ - 记录，与清洁人员分享，必要时重新培训

消息模板最佳实践

模板创建： - 🎯 保持简短：理想150-300字 - 🎯 使用变量：个性化每条消息 - 🎯 测试变量：先发给自己验证替换 - 🎯 仔细校对：模板多次使用，错误会倍增 - 🎯 少量使用表情符号：一两个用于视觉分隔，不要过多

模板使用： - 🎯 需要时自定义：针对特定情况在发送前编辑模板 - 🎯 不要过度使用：平衡模板和个人消息 - 🎯 定期更新：随着房产变化刷新内容 - 🎯 跟踪效果：监控客人响应以了解什么有效

模板组织： - 🎯 命名一致：“入住 - [场景]”格式 - 🎯 正确分类：使查找模板更快 - 🎯 删除未使用的：保持列表可管理（最多10-15个模板） - 🎯 按频率排序：最常用的模板在顶部

变量使用： - 🎯 必需变量优先：{{guest_name}}、{{listing_title}} - 🎯 可选变量：在模板中添加优雅的备用方案 - 🎯 测试边缘情况：如果WiFi密码缺失怎么办？ - 🎯 保持变量更新：更改了门锁密码？更新模板！

智能定价最佳实践

时长折扣： - 🎯 保守开始：10%周折扣、20%月折扣 - 🎯 监控入住率：如果<70%，增加折扣 - 🎯 季节性调整：淡季更高折扣 - 🎯 竞争分析：检查附近房源的折扣 - 🎯 测试与迭代：尝试不同百分比，衡量结果 - 🎯 跟踪预订模式：了解客人何时预订更长住宿 - 🎯 计算盈亏平衡：了解盈利所需的最低入住率

即时优惠： - 🎯 低入住率期间启用：不在旺季 - 🎯 设置30-40%折扣：足够大以激励今晚预订 - 🎯 不要过度使用：培养客人等待优惠的习惯 - 🎯 结合即时预订：更快的预订流程 - 🎯 跟踪转化：多少优惠带来预订？ - 🎯 周末重点：周五/周六晚更有效 - 🎯 监控客人质量：确保即时优惠客人维持标准

一般策略： - 🎯 首先正确设置基础价格：折扣适用于基础价格 - 🎯 使用定价预览：保存前验证价格 - 🎯 季度审核：根据预订模式调整 - 🎯 不要过度折扣：50%以上折扣损害感知价值 - 🎯 传达价值：客人喜欢”获得优惠” - 🎯 A/B测试：尝试不同折扣级别30天 - 🎯 市场定位：相对于可比房产定价

业务策略：

何时使用时长折扣： - 低入住率（<70%） - 淡季月份 - 竞争激烈的市场 - 房产受益于长住（减少周转）

何时不使用时长折扣： - 旺季（需求已经很高） - 无竞争的独特房产 - 已达90%以上入住率

何时使用即时优惠： - 今晚日历空置 - 城市位置（商务旅行者） - 高清洁成本（通过一些收入抵消）

何时不使用即时优惠： - 旺季假期（无论如何都会预订） - 偏远位置（无法当天预订） - 房产需要提前准备时间

投资回报率计算：

确定智能定价是否有效：

  1. 跟踪入住率（启用前后）
  2. 计算平均每晚收入（总收入 / 预订晚数）
  3. 衡量总月收入（入住率 × 平均价格 × 天数）
  4. 比较期间（前后3个月）

成功指标： - 入住率增长：+10-20% - 总收入增长：+15-30% - 平均预订时长：+50-100% - 客人满意度：保持或提高评分

联合托管最佳实践

对于房东： - 🎯 仔细审核联合房东：检查推荐信、过往经验 - 🎯 设定明确期望：响应时间、沟通风格 - 🎯 定期审核活动日志：确保质量控制 - 🎯 提供培训：分享您的托管理念和标准 - 🎯 认可出色工作：感谢和奖励表现优秀者 - 🎯 使用分析：数据驱动的团队管理决策 - 🎯 清洁人员可用性：确保繁忙期间有备用清洁人员 - 🎯 记录流程：创建标准操作程序 - 🎯 定期检查：每周或每月团队会议 - 🎯 绩效评估：季度团队成员评估

对于联合房东： - 🎯 快速响应：清醒时间<2小时 - 🎯 匹配房东风格：保持一致的品牌声音 - 🎯 记录所有变更：通过活动日志通知房东 - 🎯 不要越权：尊重定价/付款限制 - 🎯 沟通问题：立即向房东报告问题 - 🎯 主动出击：预见客人需求 - 🎯 了解房产细节：了解设施、特点、说明 - 🎯 明智使用模板：需要时个性化 - 🎯 跟踪您的指标：监控响应时间、客人满意度

对于清洁人员： - 🎯 更新可用性：不可用时阻止日期 - 🎯 及时完成任务：在下次入住前 - 🎯 上传照片：高质量工作的证明 - 🎯 立即报告问题：不要等到下一个任务 - 🎯 保持标准：遵循房东的清洁清单 - 🎯 补充用品：跟踪库存，请求补货 - 🎯 尊重客人隐私：如果客人在场不要进入 - 🎯 保护房产安全：锁门、关窗、设置警报

附录 H: 自动重排详解

自动重排：智能预订修改

自动重排在几秒钟内自动处理预订变更。

自动重排处理什么

  1. 客人发起的变更：
    • 日期延长或缩短
    • 入住/退房时间修改
    • 客人数量变更
    • 提前入住请求
    • 延迟退房请求
  2. 房东发起的变更：
    • 批准提前入住/延迟退房
    • 手动调整预订日期
    • 修改客人数量
  3. 取消：
    • 自动清理相关任务
    • 日历重新开放
    • 可选即时优惠触发

自动更新内容

1. 清洁任务 - 根据新退房时间重新计算清洁开始/结束时间 - 如果原清洁人员有冲突则重新分配 - 保留前后照片要求 - 向清洁人员发送更新时间表的通知

2. 日历与缓冲 - 立即阻止延长的晚上 - 释放取消的晚上供新预订 - 如果周转时间 < 3小时则插入智能缓冲（可配置） - 如果SLA受威胁则标记”紧张周转”风险

3. 联合房东值班时段 - 调整值班时间以匹配新日期 - 更新消息路由（客人看到正确的值班联合房东） - 协调联合房东之间的交接

4. 客人消息 - 通过模板自动发送变更确认 - 更新新时间的入住/退房说明 - 包括任何新的入门密码或特殊说明

5. 即时定价（可选） - 如果72小时内释放晚上，建议启用即时优惠 - 提高可见度以填补意外空缺

示例场景

场景 1：客人请求日期延长 - 客人要求延长2晚 - 系统检查可用性 - 自动重排更新清洁任务、日历和消息

场景 2：批准提前入住

预检面板显示： - 清洁人员影响： “清洁必须在上午11:30前完成（原为下午2:30）” - 风险徽章： “紧张周转”（如果前一个退房是上午11点） - 替代方案： “建议下午1点入住以留出安全缓冲？”

选择操作： - 选项 A： 确认中午12点入住（清洁人员会赶时间） - 选项 B： 反提议下午1点入住（更安全） - 选项 C： 将前一位客人的退房延迟1小时

自动重排执行： - 清洁任务调整为上午11:30前完成 - 清洁人员收到紧急通知 - 客人收到更新的说明 - 入门密码生成（如果是智能锁）

场景 3：客人数量增加 - 更新清洁任务备注（“4位客人，需要额外床单”） - 向客人发送更新的定价 - 在活动日志中记录变更

冲突解决

如果原清洁人员无法适应新时间表：

系统逻辑： 1. 检查清洁人员新时段的可用性 2. 如果不可用，按以下条件找下一个最佳清洁人员： - 可用性（主要） - 到房产的行程时间（最小化） - 评分（质量） - 当前工作量（平衡）

备用选项： - 没有清洁人员可用？ - 建议插入缓冲（延迟客人入住1-2小时） - 提供将预订推迟到下一个可用时段 - 上报房东手动解决

房东通知： - “原清洁人员不可用。已分配备用清洁人员Mike（4.9★，5英里外）” - “风险：紧张周转。考虑1小时缓冲？”

风险徽章与警告

自动重排标记潜在问题：

  • 🔴 紧张周转： 退房和入住之间<3小时（SLA风险）
  • 🟡 清洁人员重新分配： 原清洁人员不可用，已分配备用
  • 🟡 无清洁人员： 未找到可用清洁人员（需要手动干预）
  • 🔴 SLA违规： 不可能按时清洁房产

活动日志集成

所有自动重排操作都有完整归属记录：

查看活动日志： - 房源 → 活动 → 按”自动重排”筛选 - 查看所有自动调整的完整历史

客人沟通模板

通过模板自动生成发送给客人的消息：

  • 模板 1：日期延长确认
  • 模板 2：提前入住批准

模板可编辑： - 在设置 → 消息模板中自定义默认消息 - 变量自动填充（{客人姓名}、{日期}等）

最佳实践

  • 确认变更前审核预检面板
  • 设置最小周转缓冲（建议3-4小时）
  • 启用清洁人员确认的推送通知
  • 每周检查活动日志以发现模式
  • 与清洁人员沟通他们的可用性偏好
  • 没有清洁人员确认不要批准紧张周转
  • 旺季延长预订前检查日历冲突

配置选项

可调设置： - 最小周转缓冲： 2-6小时（默认：3小时） - 自动分配备用清洁人员： 开/关（默认：开） - 需要清洁人员确认： 清洁前<2小时的变更（默认：开） - 客人消息模板： 自定义确认消息 - 风险阈值： 何时显示警告（保守 / 平衡 / 激进）

与团队管理的集成

联合房东权限： - 可以批准客人请求的修改 - 查看清洁人员分配和风险徽章 - 无法覆盖SLA警告（需要房东批准）

清洁人员通知： - 时间表变更时推送通知 - 深层链接到更新的任务详情 - 如果不可用可以拒绝的选项（触发重新分配）

活动归属： - 所有变更显示谁批准的（房东 vs 联合房东姓名） - 清洁人员可以查看其任务的修改历史

未来增强

  • 多房产负载均衡： 在整个投资组合中自动分配清洁人员
  • 外部日历同步： 与Airbnb/VRBO预订变更协调
  • 价格优化： 自动建议释放晚上的最优定价
  • 预测调度： AI预测可能的修改请求并预先规划替代方案

CN Release 1.0 费用模式

重要： CN Release 1.0 的费用结构与之前版本不同。

费用结构

方 费用
客人 0%服务费 – 客人不支付任何平台费用
房东 等级相关平台费 – 见下表

房东等级与平台费

更高等级的房东支付更低的平台费：

房东等级 年收入 平台费 福利
银牌 $0+ 13% 标准支持

各等级示例

银牌等级（13%费用）： - 预订总额：$1,000 - 平台费：13% = $130 - 房东收款：$870

金牌等级（12.5%费用）： - 预订总额：$1,000 - 平台费：12.5% = $125 - 房东收款：$875

白金等级（12%费用）： - 预订总额：$1,000 - 平台费：12% = $120 - 房东收款：$880

钻石等级（11.5%费用）： - 预订总额：$1,000 - 平台费：11.5% = $115 - 房东收款：$885

等级晋升

您的等级基于年收入（每年重置）： - 银牌：$0+（默认） - 金牌：$50,000+ - 白金：$250,000+ - 钻石：$500,000+

重要说明

  • 更高等级 = 更低平台费 = 更高收款
  • 等级根据过去12个月收入自动计算
  • 费用变更仅适用于新预订（现有预订保持原费用）
  • 在个人资料 → 房东仪表板中查看您当前等级

技术：平台费如何计算

平台费使用简单公式，带有管理员可配置的值：

房东平台费 = platform_fee_rate - tier_discount_rate
配置值 默认 描述
platform_fee_rate 15% 基础平台费（所有人相同）
host_silver_rate 2% 银牌等级折扣
host_gold_rate 2.5% 金牌等级折扣
host_platinum_rate 3% 白金等级折扣
host_diamond_rate 3.5% 钻石等级折扣

结果： - 银牌：15% - 2% = 13% - 金牌：15% - 2.5% = 12.5% - 白金：15% - 3% = 12% - 钻石：15% - 3.5% = 11.5%

对于管理员： 这些值存储在 unified_config 数据库表中，可以通过管理员仪表板（LoyaltyConfigView）修改，无需代码更改。

附录 K: 取消与退款政策（CN Release 1.0）

本附录解释您可以为房源设置的取消政策。

可用取消政策

创建或编辑房源时，您选择以下政策之一：

灵活政策

最适合： 竞争激烈市场中的房产，吸引更多预订

客人取消时间 客人退款 您的收款
入住前24小时以上 100%退款 $0
入住前不到24小时 无退款 全额收款

适中政策

最适合： 灵活性与承诺之间的平衡

客人取消时间 客人退款 您的收款
入住前5天以上 100%退款 $0
入住前2-5天 50%退款 50%收款
入住前不到2天 无退款 全额收款

严格政策

最适合： 高需求房产、季节性租赁

客人取消时间 客人退款 您的收款
入住前7天以上 100%退款 $0
入住前7-30天 50%退款 50%收款
入住前不到7天 无退款 全额收款

房东发起的取消

警告： 作为房东取消预订有严重后果：

  • 客人获得全额退款包括所有费用
  • 您可能被收取取消费
  • 您的取消率被跟踪和显示
  • 高取消率可能导致：
    • 搜索排名下降
    • 失去超级房东身份
    • 账户暂停

您可以取消的情况： - 客人违反房屋规则 - 前一位客人造成的房产损坏 - 需要紧急维修 - 不可抗力事件

取消前务必联系支持以记录合法原因。

设置您的取消政策

  1. 进入房源标签
  2. 点击要编辑的房源
  3. 滚动到政策部分
  4. 选择取消政策
  5. 选择灵活、适中或严格
  6. 点击保存

政策变更仅适用于新预订，不适用于现有预订。

房东提示

  • ✅ 灵活吸引更多预订但也更多取消
  • ✅ 严格提供收入保障但可能减少预订
  • ✅ 适中推荐给大多数房东
  • ✅ 考虑季节性政策（高需求期更严格）
