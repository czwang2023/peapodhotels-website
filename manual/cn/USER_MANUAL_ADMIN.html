8. 管理员指南

8.1 管理员概述

↑ 返回目录

作为管理员，您：

  • 👥 管理用户 - 查看、搜索、消息、暂停或删除账户
  • ⚙️ 配置应用设置 - 忠诚度计划、费用、功能开关
  • 📊 查看分析 - 平台范围的指标和趋势
  • 📋 查看报告 - 平台健康、API指标、数据库使用
  • 🚨 处理问题 - 争议、违规、紧急情况

重要： 管理员不创建或编辑房源。只有房东创建房源，审核员批准，管理员可以查看平台分析。

8.3 工单（Bug报告和反馈）

↑ 返回目录

访问： 底部导航 → “工单”标签

两个类别： 1. Bugs - 用户报告的错误和问题 2. 反馈 - 用户建议和功能请求

Bug报告详情：

  • 标题 - 问题描述
  • 优先级 - 低/中/高/紧急（带颜色指示器）
  • 状态 - 新/未分配/处理中/已解决/已关闭
  • 报告者 - 报告的用户邮箱
  • 日期 - “X天前”
  • 徽章 - 未读报告显示”新”

操作：

  • 点击报告查看完整详情
  • 分配给团队成员
  • 更新状态
  • 添加备注/评论
  • 标记为已解决
  • 通知用户解决方案

反馈管理：

  • 类似Bug报告
  • 类别：功能请求、UX改进、其他
  • 投票数（其他用户的赞成票）
  • 实施状态跟踪

8.4 用户管理

↑ 返回目录

访问： 管理员仪表板 → “用户”

用户列表显示：

  • 头像
  • 全名
  • 邮箱
  • 用户角色（客人、房东、审核员、管理员）
  • 加入日期
  • 账户状态（活跃、已暂停、已删除）
  • 验证状态

用户详情页面：

点击列表中任何用户访问用户详情。通过多个标签查看综合信息：

  • 概览
  • 预订（如果是客人）
  • 房源（如果是房东）
  • 评价
  • 消息
  • 付款历史
  • 活动日志

管理员操作：

  • 发送消息
  • 重置密码
  • 暂停账户（7/30天或永久）
  • 删除账户

向用户发送消息：

从用户详情页面向用户发送直接消息。用户在收件箱中收到消息。

重置用户密码：

向用户邮箱发送密码重置链接。可从用户详情页面访问。

暂停用户：

临时或永久限制用户访问。选项包括：

  • 原因： 违反条款、欺诈活动、垃圾邮件、安全顾虑、其他
  • 持续时间： 7天、30天、90天或永久
  • 暂停期间用户无法登录
  • 用户收到邮件通知
  • 包含供管理员记录的内部备注

删除用户：

永久删除用户账户。此操作：

  • 无法撤销
  • 需要邮箱确认以防止意外删除
  • 归档所有用户数据
  • 永久从平台删除用户

  • 金牌（¥2,500+/年）

  • 白金（¥5,000+/年）
    • 年消费门槛：¥5,000

8.5.2 推荐计划配置

访问： 配置标签 → “推荐”子标签

概述： 配置客人推荐系统 - 朋友预订时客人赚取多少佣金，以及自定义分享消息。

配置部分：

A. 客人推荐设置

佣金率：

  • 标签： “佣金率”
  • 默认： 预订金额的2.0%
  • 范围： 0% - 10%（建议：1-3%）
  • 示例： 2%意味着推荐人每¥100预订赚取200积分

持续时间：

  • 标签： “推荐持续时间”
  • 默认： 12个月
  • 范围： 1-24个月
  • 描述： 推荐人从被推荐人预订中赚取佣金的时长

积分转换：

  • 标签： “积分兑美元率”
  • 默认： 100（100积分 = ¥1.00）
  • 固定： 不可更改（保持一致性）

B. 分享消息模板

分享链接模板：

  • 标签： “分享链接模板”
  • 默认： https://peapodhotels.com/join?ref={CODE}
  • 占位符： {CODE} - 替换为推荐码
  • 示例： https://peapodhotels.com/join?ref=JOHN1234

分享消息模板：

  • 标签： “分享消息模板”

  • 默认：

    嘿！我在用 Peapod 找超棒的住处。
用我的推荐码 {CODE} 加入我，开始探索吧！

{LINK}

  • 占位符：

    • {CODE} - 用户的推荐码
    • {NAME} - 用户全名
    • {LINK} - 生成的邀请链接

  • 字符限制： 建议500字符

  • 多行： 支持换行

分享如何工作： 1. 客人在推荐标签点击”分享” 2. 应用从配置获取模板 3. 用实际值替换占位符 4. 打开iOS分享表单 5. 客人通过iMessage、WhatsApp、邮件等分享

示例自定义：

激进活动（3%）：

佣金率：3.0%
持续时间：12个月
消息："免费获得¥30！使用我的推荐码 {CODE}，我们都省钱！"

保守（1%）：

佣金率：1.0%
持续时间：6个月
消息："我爱 Peapod！用 {CODE} 加入"

重要说明：

  • 更改立即适用于所有新推荐
  • 现有推荐关系保持原始佣金%
  • 分享消息更改立即影响所有用户
  • 在宣布前用客人账户测试更改

8.5.3 费用与定价配置

平台费用结构（CN Release 1.0）：

⚠️ 重要： CN Release 1.0 使用仅房东费用模式以吸引客人。

当前费用结构：

方 费用 说明
客人服务费 0% 客人不支付任何平台费用
房东平台费 等级相关 见下表
最低预订费 ¥5（固定） 适用于非常小的预订
支付处理 2.9% + ¥0.30 Stripe标准（平台吸收）

房东等级费用（CN Release 1.0）：

更高等级的房东支付更低的平台费：

房东等级 年收入 平台费 福利
银牌 ¥0+ 13% 标准支持
金牌 ¥50,000+ 12.5% 优先支持、精选展示
白金 ¥250,000+ 12% 高级支持、网络研讨会、提前获取功能
钻石 ¥500,000+ 11.5% VIP 24/7、客户经理、定制营销

房东等级费用如何工作： 1. 房东完成预订，赚取毛收入 2. 根据房东当前等级应用平台费 3. 房东收到收款（预订减去等级费用） 4. 等级根据过去12个月收入重新计算

各等级示例收款（¥1,000预订）：

等级 费用 收款
银牌 13%（¥130） ¥870
金牌 12.5%（¥125） ¥875
白金 12%（¥120） ¥880
钻石 11.5%（¥115） ¥885

区域定价：

  • 美国市场： USD、Stripe支付
  • 中国市场： CNY、微信/支付宝（未来）
  • 货币转换： 实时汇率 + 2%加成

可编辑字段：

  • 客人服务费百分比（目前CN Release为0%）
  • 每个等级的房东平台费（银牌13%、金牌12.5%、白金12%、钻石11.5%）
  • 等级收入门槛（¥0、¥50k、¥250k、¥500k）
  • 最低预订费金额
  • 货币转换加成
  • 区域定价规则

8.8 徽章管理

↑ 返回目录

什么是徽章管理？

管理员界面用于创建、编辑、批准和撤销房东质量徽章和房产功能徽章。确保徽章完整性和质量标准。

访问： 配置标签 → “徽章”部分（或管理员仪表板 → 徽章管理）

徽章类型： 1. 房东质量徽章 - 基于绩效（超级房东、即时回复、顶级房东） 2. 房产功能徽章 - 已验证功能（可持续住宿、远程工作友好、宠物友好）

8.8.1 查看所有徽章

通过配置 → “徽章管理”访问徽章管理。两个类别可用：

房东徽章（自动化）：

  • 超级房东
  • 即时回复
  • 经验丰富的房东
  • 新房东
  • 顶级房东

房产徽章（需要验证）：

  • 可持续住宿
  • 远程工作友好
  • 宠物友好认证
  • 家庭友好认证
  • 豪华住宿

徽章信息显示：

  • 徽章图标和名称
  • 类型：自动化或需要验证
  • 状态：活跃/非活跃
  • 授予总数
  • 待申请数（验证徽章）
  • 操作：编辑/查看详情/停用

8.8.2 审核徽章申请

对于管理员验证的徽章（可持续、远程工作、豪华）：

审核房东提交的房产徽章申请。每个申请包括：

  • 房东姓名和房产名称
  • 申请日期
  • 状态：待审核/已批准/已拒绝
  • 提交的文档（照片和描述）

审核流程：

审核提交的照片和文档以验证声明：

  • 检查照片质量和真实性
  • 验证功能符合徽章标准
  • 寻找危险信号（库存照片、不清晰图像、缺少文档）

决定操作：

批准徽章：

  • 授予房产徽章
  • 徽章立即显示在房源上
  • 房东收到批准通知
  • 徽章显示在搜索结果中

请求更多信息：

  • 请求额外文档或更清晰的照片
  • 房东收到反馈通知
  • 申请状态更新为”请求更多信息”
  • 房东可以重新提交改进的文档

拒绝徽章：

  • 拒绝徽章申请
  • 选择拒绝原因（文档不足、不符合标准、假照片、不符合资格）
  • 提供可选反馈消息
  • 房东收到拒绝通知
  • 申请归档（30天后可重新申请）

8.8.3 撤销徽章

何时撤销：

  • 客人投诉功能不符
  • 房东移除功能（例如断开太阳能板）
  • 房产不再符合标准
  • 检测到徽章欺诈

撤销流程：

搜索有问题徽章的房产。查看徽章授予详情包括：

  • 授予日期
  • 验证照片
  • 原始文档
  • 最近的投诉或问题

撤销徽章：

  • 选择撤销原因（功能不再符合标准、虚假陈述、欺诈等）
  • 向房东提供解释撤销的消息
  • 确认撤销
  • 徽章立即从房源删除
  • 房东收到通知
  • 如适用房产标记重新验证

8.8.4 自动化徽章监控

超级房东自动审核（季度）：

发生什么：

  • 系统在1月1日、4月1日、7月1日、10月1日季度运行
  • 检查所有房东是否符合超级房东标准：
    • 4.8+评分
    • 90%+响应率
    • <1%取消率
    • 过去12个月10+完成预订
  • 授予徽章给符合资格的房东
  • 从不再符合资格的房东移除徽章
  • 向受影响房东发送通知

管理员监控：

通过徽章管理 → 超级房东查看自动审核计划和结果。审核包括：

  • 即将到来的审核日期
  • 上次审核结果（授予的房东、失去徽章的房东、失去原因）
  • 详细变更日志

手动覆盖：

  • 手动授予超级房东给值得的房东（边缘情况）
  • 因违规手动撤销（欺诈、重大问题）
  • 所有覆盖记录在审计日志中

8.8.5 徽章统计与报告

徽章分析仪表板：

通过徽章管理 → 分析标签访问。查看关键指标：

  • 授予徽章总数：总数
  • 超级房东数量：活跃房东百分比
  • 待申请：当前队列
  • 撤销：最近的删除

徽章绩效指标：

  • 点击率：徽章多频繁导致房源查看
  • 转化率：有徽章房源的预订vs无徽章
  • 申请量：每种徽章类型的申请
  • 批准率：批准的申请百分比
  • 撤销率：撤销的徽章百分比

示例徽章报告：

可持续住宿徽章指标包括：

  • 授予的房产总数
  • 待申请
  • 批准率（批准/拒绝/待审核分解）
  • 撤销率
  • 转化提升（预订vs非认证）
  • 平均定价溢价

导出选项：

  • CSV：带授予日期的徽章列表
  • PDF：月度徽章报告
  • JSON：用于外部分析的API数据

8.9 消息模板管理

↑ 返回目录

什么是模板管理？

管理员界面用于管理所有房东可用的默认消息模板。编辑内容、添加新模板或在平台范围删除模板。

访问： 配置标签 → “消息模板”部分

8.9.1 查看默认模板

通过配置 → “消息模板”访问查看10个默认模板：

  • 入住说明（详细版）
  • 入住说明（简版）
  • 提前入住请求
  • 房屋规则提醒
  • 安静时间提醒
  • 本地推荐
  • 交通提示
  • 退房感谢
  • 问题解决
  • 定价明细说明

模板信息：

  • 名称
  • 类别（入住、规则、提示、通用、定价）
  • 使用次数（平台范围）
  • 最后修改日期
  • 状态：活跃/非活跃
  • 操作：编辑/预览/停用/删除

8.9.2 编辑默认模板

选择任何模板打开编辑器。编辑模板字段：

  • 名称：模板名称（可编辑）
  • 类别：下拉菜单（入住、规则、提示、通用、定价）
  • 内容：多行文本编辑器（500字符限制）
  • 变量：可用变量（{{guest_name}}、{{listing_title}}等）
  • 使用次数：平台范围使用（只读）

编辑流程： 1. 编辑模板内容和名称 2. 使用”插入变量”添加动态占位符 3. 使用示例数据预览模板 4. 保存更改 5. 更新的模板立即对所有房东可用

重要说明：

  • 更改适用于平台范围所有房东
  • 自定义了此模板的房东保留他们的自定义版本
  • 仅影响默认模板（房东仍可单独编辑）

8.9.3 创建新默认模板

通过配置 → 消息模板 → “+ 新模板”创建新模板

模板创建： 1. 填写必需字段：

  • 名称：模板名称
  • 类别：选择类别
  • 内容：带变量的模板文本
  1. 使用”插入变量”按钮在光标位置插入变量
  2. 使用示例数据预览模板
  3. 保存模板
  4. 新模板立即对所有房东可用

可用变量（25+）：

  • 客人：{{guest_name}}、{{guest_full_name}}、{{guest_count}}
  • 房产：{{listing_title}}、{{property_address}}、{{wifi_password}}、{{lockbox_code}}
  • 预订：{{check_in_date}}、{{check_in_time}}、{{check_out_date}}、{{nights}}
  • 房东：{{host_name}}、{{host_phone}}
  • 定价：{{nightly_rate}}、{{total_price}}、{{cleaning_fee}}

8.9.4 停用/删除模板

停用模板：

  • 从房东UI隐藏模板
  • 房东无法用它发送新消息
  • 现有自定义版本仍有效
  • 可以稍后重新激活
  • 用于：季节性模板、弃用模板

停用流程： 选择模板，点击”停用”，确认操作。模板状态变为”非活跃”，房东在模板选择器中不再看到它。

删除模板：

  • 永久删除模板
  • 房东失去访问（除非他们自定义了）
  • 无法撤销
  • 用于：冗余模板、政策违规

删除流程： 选择模板，点击”删除”，输入确认（“DELETE”），确认。模板从数据库永久删除。

8.9.5 模板使用分析

通过配置 → 消息模板 → 分析访问分析

使用仪表板指标：

  • 最常用模板：按使用次数排名第一
  • 最少使用：使用率低的模板
  • 模板发送总数：已发送消息（过去30天）
  • 使用模板的房东：采用率百分比
  • 节省时间：估计效率提升

模板绩效：

查看按使用率排名的热门模板： 1. 最受欢迎模板（高使用率） 2. 中等使用模板 3. 很少使用模板（删除候选）

洞察：

  • 识别热门模板（保留和改进）
  • 识别未使用模板（考虑删除）
  • 衡量采用率（使用模板的房东百分比）
  • 计算节省时间（发送消息 × 平均节省时间）

8.10 评价审核

↑ 返回目录

什么是评价审核？

管理员工具用于审核标记的内容、审核照片评价和维护评价质量标准。

访问： 管理员仪表板 → “评价审核”或配置 → “评价”

8.10.1 评价审核队列

通过管理员 → 评价审核访问。三个队列标签可用：

  • 标记评价 - 用户报告的不当内容
  • 照片评价 - 等待批准的新照片评价
  • 垃圾检测 - 算法标记的潜在垃圾

标记评价信息：

  • 评价者姓名和头像
  • 评价文本（完整内容）
  • 照片（如包含）
  • 星级评分
  • 房产名称和房东
  • 标记原因：“不当语言”/“虚假评价”/“垃圾”/“骚扰”
  • 报告者：谁标记的
  • 标记日期
  • 操作：批准/编辑/删除

8.10.2 审核标记评价

审核标记内容并根据社区准则采取适当行动。

批准（保留）评价：

  • 评价不违反准则
  • 点击”批准”按钮并确认
  • 评价保持发布
  • 标记驳回
  • 报告者收到决定通知

删除评价：

  • 评价违反社区准则（脏话、人身攻击、垃圾、离题）
  • 点击”删除”按钮
  • 选择违规类型：
    • 脏话/冒犯性语言
    • 人身攻击/骚扰
    • 垃圾/虚假评价
    • 离题内容
    • 其他（指定）
  • 确认删除
  • 评价从房源删除
  • 评价者和房东收到通知

编辑评价（删除脏话）：

  • 评价包含轻微脏话但其他方面是有效反馈
  • 点击”编辑”按钮
  • 编辑文本删除不当内容
  • 保持评价其余部分完整
  • 保存编辑
  • 编辑后的评价发布
  • 评价者收到通知
  • 原始保存在审核日志中

8.10.3 照片评价审核

审核客人的照片提交以确保质量并符合准则。

审核照片提交：

访问审核队列中的”照片评价”标签。每个提交显示：

  • 评价者姓名
  • 房产名称
  • 星级评分
  • 评价文本
  • 上传的照片（最多5张）

审核流程：

检查每张照片：

  • 质量（清晰、光线好、相关）
  • 隐私（无可识别的人）
  • 适当性（无不当内容）
  • 真实性（非库存照片）
  • 相关性（与房产相关）

带照片删除批准：

如果一些照片违反准则：

  • 点击”带变更批准”
  • 选择要删除的照片
  • 提供原因（隐私违规、不当内容等）
  • 批准评价
  • 评价仅带批准的照片发布
  • 删除的照片被删除
  • 评价者收到通知

照片违规类型：

  • 可识别的人（隐私）
  • 不当内容（裸体、暴力）
  • 与房产无关（垃圾）
  • 库存照片（虚假评价）
  • 低质量（模糊、暗）

8.11 审核员SLA上报（CN Release 1.0）

↑ 返回目录

什么是SLA上报？

当审核员未能达到服务级别协议目标时，系统自动上报给管理员。这确保房东获得及时响应并维护平台质量。

访问： 管理员仪表板 → “SLA警报”或通知 → “审核员SLA”

8.11.1 SLA目标参考

审核类型 目标时间 最大时间 上报触发
新房源批准 24小时 48小时 > 48小时未审核
关键编辑审核 24小时 48小时 > 48小时未审核
重新提交审核 24小时 48小时 > 48小时未审核
安全/滥用报告 4小时 8小时 > 8小时未处理
争议解决 24小时 48小时 > 48小时未解决

8.11.2 SLA上报警报

您将看到：

当SLA违规时，管理员收到：

  • 🚨 推送通知： “SLA违规：[审核员姓名] - [审核类型] 待审核52小时”
  • 📧 邮件警报： 带待审核项链接的详细分解
  • 📊 仪表板徽章： “X个SLA违规”计数

警报信息：

  • 审核员姓名和个人资料
  • 审核类型（新房源、关键编辑、安全等）
  • 队列中时间（超过SLA多久）
  • 房源/房产详情
  • 房东姓名（谁受影响）

8.11.3 SLA违规时的管理员操作

选项 1：消息审核员

  • 发送消息询问状态更新
  • 审核员可能有合理原因（生病、紧急）
  • 设置后续提醒

选项 2：重新分配任务

  • 将待审核移至另一个审核员
  • 原审核员收到重新分配通知
  • 任务获得新的SLA时间

选项 3：管理员完成审核

  • 管理员接管并直接完成审核
  • 用于紧急项目或无审核员可用时
  • 记录在审计轨迹中

选项 4：延长SLA（带原因）

  • 授予临时延期（例如+24小时）
  • 需要记录的原因
  • 房东不收到通知（仅内部）

8.11.4 SLA绩效仪表板

访问： 管理员仪表板 → “审核员绩效” → “SLA合规”

显示指标：

  • 整体SLA合规率： 平台范围%（目标：95%+）
  • 本周违规： SLA违规计数
  • 本月违规： 月度总计
  • 平均审核时间： 平台范围平均

每审核员分解：

  • 审核员姓名
  • 本月审核总数
  • SLA合规率（%）
  • 违规次数
  • 平均审核时间
  • 趋势（改进/下降）

示例仪表板：

审核员 审核数 SLA率 违规 平均时间
Sarah K. 45 98% 1 8.5h
Mike L. 38 95% 2 14.2h
Jane W. 52 100% 0 6.1h
总计 135 97.5% 3 9.6h

8.11.5 SLA违规模式与行动

识别问题：

  • 单次违规：可能是一次性（疾病、紧急）
  • 3次以上/月：模式 - 需要讨论
  • 持续违规：绩效问题 - 可能需要培训或移除

建议行动：

违规频率 管理员行动
首次违规 发送友好提醒
2-3次/月 安排1对1讨论
4次以上/月 绩效改进计划
持续违规 考虑角色重新分配或移除

文档：

  • 所有SLA违规自动记录
  • 管理员行动记录在审计轨迹中
  • 审核员收到任何正式行动的副本

8.7.4 问题跟踪

  • 房源报告（安全问题、不准确）

  • 消息报告（垃圾、骚扰）

问题解决流程： 通过”报告” → “问题”访问查看未解决问题。对于每个问题： 1. 审核证据（截屏、消息） 2. 采取适当行动：

  • 驳回（无需操作）
  • 警告用户
  • 删除内容
  • 暂停用户
  1. 写解决备注
  2. 标记为已解决
  3. 报告者收到通知

8.7.5 导出用户数据

导出各种数据类型用于分析或GDPR合规：

  • 所有用户数据
  • 预订历史
  • 房源数据
  • 分析

通过”报告” → “数据导出”访问，选择数据类型和日期范围，生成导出，下载CSV文件。

8.7.6 月度报告

自动化月度平台摘要包括：

  • 收入报告
  • 用户增长报告
  • 预订趋势
  • 绩效指标

通过”报告” → “月度报告”访问，选择月份/年份，查看综合报告，下载为PDF或CSV。
