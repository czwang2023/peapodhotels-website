附录A：通勤筛选技术详情

地理编码详情

筛选器接受多种地址格式： - 完整地址：“123 Main St, San Francisco, CA 94102” - 部分地址：“Golden Gate Park, SF” - 地标：“Moscone Center” - 商家名称：“Apple Park”

地理编码支持： - 使用Apple的MapKit地理编码服务 - 将地址文本转换为GPS坐标（经纬度） - 位置成功验证时显示绿色勾号 - 无法找到地址时显示红色错误消息

地理编码故障排除： - “无法解析地址”：尝试更具体的地址或使用地标名称 - “位置已验证”未显示：输入地址后点击”设置”按钮 - 位置错误：添加更多细节重新输入地址（添加城市、州）

筛选算法

通勤时间如何计算： 1. 用户输入目的地地址 2. 系统将目的地地理编码为GPS坐标 3. 对于数据库中的每个房源： - 计算从房源到目的地的通勤时间 - 使用选定的交通方式（驾车/公交/骑行/步行） - 与最大时间阈值比较 4. 仅显示在时间限制内的房源

交通方式详情： - 驾车：使用实时交通数据的汽车路线 - 公交：使用公共交通路线（公交、地铁、火车时刻表） - 骑行：使用自行车友好路线和道路 - 步行：使用人行道

按模式持久化： 每种交通方式记住自己的时间设置： - 驾车：20分钟 - 公交：40分钟 - 骑行：30分钟 - 步行：15分钟

当您切换模式时，该模式上次使用的时间会恢复。

性能

会话记忆： 关闭应用时，您的目的地和时间设置使用UserDefaults保存：

UserDefaults.standard.set(destination, forKey: "commute_destination")
UserDefaults.standard.set(drivingTime, forKey: "commute_time_drive")
UserDefaults.standard.set(transitTime, forKey: "commute_time_transit")

示例用户流程：

第1天：设置"123 Market St, SF"为目的地
       配置驾车：20分钟，公交：40分钟
       查找并预订房源

第2天：重新打开应用 - 设置自动恢复！
       相同目的地，相同时间限制
       无需重新输入即可继续搜索

限制

已知限制： - 通勤时间是估计值，可能随交通状况变化 - 公交时间假设服务正在运行（请查看时刻表） - 骑行时间假设中等体能水平 - 步行时间假设平均步行速度（3-4英里/小时） - 偏远地区可能有限的公交/骑行路线数据

最佳实践： - 按模式设置合理时间（20分钟驾车≠20分钟步行！） - 使用地标更容易输入（“Salesforce大楼”而非完整地址） - 比较驾车和公交模式（城市中公交往往更快） - 如果没有结果，增加最大时间 - 结合价格筛选获得实惠又方便的选择

附录B：地图视图技术详情

搜索圆圈架构

视觉反馈： - 粉色圆圈覆盖：显示您的活动搜索半径 - 圆圈中心：您的搜索位置（图钉图标） - 动态大小：调整半径时圆圈增大/缩小 - 自动缩放：半径变化时地图缩放以适应整个圆圈

半径控制： - 范围：5-50英里（8-80公里） - 默认：25英里（40公里） - 可调整：通过内联滑块 - 持久化：保存在UserDefaults（应用重启后保留）

理解搜索圆圈： - 粉色圆圈覆盖显示您的活动搜索半径 - 默认半径：25英里 - 圆圈中心标记您的搜索位置 - 仅显示圆圈内的房源 - 地图自动缩放以适应整个圆圈

内联半径控制

变化内容（v0.14.7）： - 之前：半径控制隐藏在筛选面板中（调整需要4次点击） - 之后：内联滑块直接在地图上（即时调整）

滑块界面： - 位置：地图视图底部（在房源卡片上方） - 设计：带值标签的水平滑块 - 范围：5英里 ←●─────→ 50英里 - 值显示：“25英里”实时更新 - 交互：拖动滑块，圆圈立即更新 - 持久化：偏好保存到UserDefaults

如何调整半径： 1. 打开地图视图 2. 找到底部显示当前半径的内联滑块（如”25英里”） 3. 向左拖动滑块减小半径： - 圆圈实时缩小 - 新半径外的房源消失 - 结果数量更新 4. 向右拖动滑块增加半径： - 圆圈扩大 - 更多房源出现 - 结果数量增加 5. 释放滑块 - 地图更新为新结果

房源卡片上的距离显示

距离徽章设计： - 位置：房源卡片右上角 - 格式：“X.X英里”或”X.X公里” - 背景：白色带阴影 - 字体：系统字体，12pt，半粗体 - 精度：1位小数（如”2.4英里”，不是”2.37英里”）

距离何时显示： - ✅ 地图视图（始终可见） - ✅ 列表视图（如果使用附近搜索或”距离最近”排序活动） - ❌ 无位置上下文的列表视图（仅按城市名搜索）

距离计算：

// 精确地理距离的Haversine公式
func distance(from: CLLocationCoordinate2D, to: CLLocationCoordinate2D) -> Double {
    let earthRadius = 3959.0 // 英里（公里用6371.0）
    let dLat = (to.latitude - from.latitude) * .pi / 180.0
    let dLon = (to.longitude - from.longitude) * .pi / 180.0

    let a = sin(dLat/2) * sin(dLat/2) +
            cos(from.latitude * .pi / 180.0) * cos(to.latitude * .pi / 180.0) *
            sin(dLon/2) * sin(dLon/2)

    let c = 2 * atan2(sqrt(a), sqrt(1-a))
    return earthRadius * c
}

    • 清晰的街道地图

PostGIS后端集成

后端筛选优势： - PostgreSQL PostGIS扩展在服务器上执行圆圈筛选 - 减少30-40%的数据传输（仅发送半径内的房源到客户端） - 精确的地理距离计算（非近似值） - 使用数据库空间索引（GIST）的更快性能

v0.14.7之前（前端筛选）：

客户端 → "获取圣地亚哥所有房源" → 服务器
服务器 → 发送10,000个房源 → 客户端
客户端 → 将10,000个筛选到25英里半径内的142个
结果：传输9,858个房源但被丢弃（浪费带宽）

v0.14.7之后（后端筛选）：

客户端 → "获取(32.7157°N, 117.1611°W)25英里内的房源" → 服务器
服务器 → PostGIS ST_DWithin()查询 → 筛选到142个房源
服务器 → 仅发送142个房源 → 客户端
结果：减少30-40%数据，加载更快

数据库查询（PostGIS ST_DWithin）：

-- 迁移061：添加PostGIS扩展
CREATE EXTENSION IF NOT EXISTS postgis;

-- 查询半径内的房源（米）
SELECT *
FROM listings
WHERE ST_DWithin(
    geography(ST_MakePoint(longitude, latitude)),
    geography(ST_MakePoint(-117.1611, 32.7157)),
    40233.6  -- 25英里转换为米
);

-- 性能：使用GIST空间索引
CREATE INDEX idx_listings_geog ON listings
USING GIST (geography(ST_MakePoint(longitude, latitude)));

性能优势： - ⚡ 地图加载速度快30-40%（网络传输数据更少） - ⚡ 精确距离（地理测量，非欧几里得近似） - ⚡ 可扩展（高效处理数百万房源） - ⚡ 节省电池（减少客户端处理）

附录D：取消与退款政策（中国版1.0）

本附录解释Peapod上可用的取消政策。当您预订房源时，房东选择的取消政策在您确认预订前显示。

理解取消政策

房东为其房源选择以下取消政策之一：

灵活政策

最适合： 需要最大灵活性的旅客

取消时间 退款金额
入住前24小时+ 100%退款
入住前不足24小时 无退款

中等政策

最适合： 计划相对确定的旅客

取消时间 退款金额
入住前5天+ 100%退款
入住前2-5天 50%退款
入住前不足2天 无退款

严格政策

最适合： 提前承诺的预算旅客

取消时间 退款金额
入住前7天+ 100%退款
入住前7-30天 50%退款
入住前不足7天 无退款

如何取消预订

  1. 前往行程标签
  2. 选择要取消的预订
  3. 点击取消预订
  4. 根据房源的取消政策查看退款金额
  5. 确认取消
  6. 退款自动处理

退款处理时间

  • 信用卡/借记卡： 5-10个工作日
  • Apple Pay： 5-10个工作日
  • 银行账户： 5-10个工作日

退款发放到预订使用的原始支付方式。

服务费

  • 服务费仅在取消导致住宿费全额退款时才退款
  • 部分退款不包括服务费退款

房东取消

在正常取消政策之外，以下情况可能提供全额退款： - 影响房源或您旅行的自然灾害 - 政府强制的旅行限制 - 严重疾病或死亡（需要文件证明） - 其他有文件记录的紧急情况
