👥 关于本手册：

本手册涵盖房东（房源所有者）和联合房东（管理日常运营的团队成员）。

联合房东角色： 负责大部分房源管理职责，包括房客沟通、预订管理、日历更新、房源维护和协调清洁。他们可以访问所有运营功能以确保日常运营顺畅。

联合房东权限： ✅ 编辑房源（描述、照片、设施、规则）、为指定房源与房客沟通、更新日历和可用性、查看和管理预订、与保洁员协调、报告问题

联合房东限制： 联合房东不能更改价格、访问收款/财务信息、邀请其他团队成员或删除房源。这些仍然是房东专属功能。

5. 房东与联合房东

5.1 导航

↑ 返回目录

房东界面使用5标签底部导航栏。每个标签提供关键房东功能的访问。

5.1.1 仪表板标签

一目了然的绩效概览。监控收入、预订和入住率趋势。

关键指标（顶部卡片）： 已收款项（已转入银行）、待收入（入住后等待收款）、本月总预订（已确认数量）、本月当前入住率（已预订晚数%）

收入概览图表： 按月显示收入的柱状图，查看最近6个月，Y轴显示人民币金额（¥0k-¥4k），点击柱状图查看精确金额

入住率部分： 本月（当月入住率%，红色）、下月（下月入住率%，灰色）- 点击”查看日历”查看详细可用性

5.1.2 房源标签

房源标签现在包含两个子标签：房源（房源管理）和任务（清洁任务管理）。

房源子标签

在一个屏幕上管理所有房源。创建新房源、编辑现有房源并监控状态。

搜索与添加： 搜索栏按名称查找 + 粉色”+“按钮（右侧）创建新房源

筛选标签： 全部、待处理编辑、待审核、请求编辑、上线、您暂停、被暂停

房源卡片显示： 房源照片、标题、价格、位置、评分/评价数、状态徽章（绿色上线/黄色待处理编辑/蓝色待审核/橙色请求编辑/红色被暂停）、三点菜单

房源子标签：搜索栏带粉色添加按钮、状态筛选、房源列表

任务子标签

查看和管理所有房源的清洁任务。监控任务进度和保洁员工作状态。

任务概览卡片： 待处理（橙色）、进行中（蓝色）、今日完成（绿色）

筛选标签： 全部、待处理、进行中、已完成

任务卡片显示： 房源名称、状态徽章、进度条（如12/20）、优先级、截止时间、保洁员姓名、任务说明

任务子标签：任务概览卡片、状态筛选、任务列表带进度条

5.1.3 预订标签

查看和管理所有房源的预订。追踪入住、收款和房客信息。

搜索与筛选： 搜索栏按房客姓名/房源/确认号查找 | 筛选标签：待处理、即将到来、已完成、全部

预订卡片显示： 房源照片、房源名称、旅客姓名、状态徽章（绿色已确认/橙色待确认）、入住/退房日期、旅客数量、收款金额（绿色）、确认号、三点菜单

刷新： 点击右上角刷新图标更新预订状态

5.1.4 消息标签

与房客沟通预订事宜。按房源和预订日期组织查看所有对话。

搜索： 搜索栏按房客姓名或房源查找对话

对话列表： 房客头像（圆形）、房源名称（粗体）、预订日期（灰色）、最后消息预览、时间戳（如”1小时前”）、未读指示器（数量徽章）

撰写新消息： 点击铅笔图标（右上角）开始新对话

消息组织： 最近对话在顶部，按房源分组，显示预订上下文便于参考

5.1.5 个人资料标签

您的账户设置、偏好和收款配置。在房东和房客角色之间切换。

个人资料头部：

  • 头像（圆形，点击更改）
  • 姓名（如”Jennifer Host”）
  • 角色徽章（如”房东 👑“带皇冠图标）
  • 切换角色按钮（蓝色）- 在房东和房客视图之间切换
  • 统计显示：
    • 行程 - 作为房客的总预订
    • 评价 - 收到的评价数
    • 会员年数 - 账户年龄

账户设置： 账户（邮箱/密码）、偏好设置（语言/地区/货币）、通知（邮件/短信/推送）、隐私与法律、房东收款、帮助中心

其他： 退出登录按钮（底部红色）、通知铃铛（右上角）

设置页面： 点击”偏好设置”打开 → 语言（简体中文/English）、地区（中国/美国）、货币（人民币¥/美元$）→ 自动保存

5.2 创建房源 [仅限房东]

↑ 返回目录

房源编辑器使用8格导航网格，您可以按任意顺序跳转各部分。每个完成的部分显示绿色勾号。

创建流程： 房源标签 → “+”按钮 → 完成8个部分 → 预览 → “提交审核”

创建前准备： 您拥有或有权出租该房源、房源符合当地法规和安全要求、有高质量照片、可提供入住说明

5.2.1 房源基础

本部分涵盖向潜在房客描述您房源的基本详情。

房源标题（40-100字符）：描述性、可搜索，突出最佳特色。示例：“La Jolla Cove附近的海滨公寓”

房源类型： 单单元（独栋房屋、公寓、客房）或多单元（酒店、青旅 - 见5.3.4）。注意： 酒店/青旅需至少一个房型/床型

描述（91-1000字符）：详细描述空间、设施、社区和独特之处。绿色计数器表示长度足够

最大房客数：使用 +/- 按钮设置容量（1-20位），影响搜索可见性和定价

房间和床位： 卧室（0-10，工作室选0）、浴室（0.5-10.0，0.5=半浴室，1.0=全浴室）、床位（1-20）

左图：房源基础顶部（标题、类型、描述）；右图：滚动后显示房客数、房间和床位设置

5.2.2 位置

输入您房源的地址和社区信息。地址验证确保地图显示的坐标准确。

地址： 街道地址（必填）、公寓/套房（可选）、城市/州/邮编（必填）- 通过地址验证自动填充

验证地址按钮： 点击”验证地址”确认位置 → 成功显示绿色勾号”✓ 地址已验证”→ 坐标自动设置

社区（必填）：描述社区、附近景点和交通，此字段显示在房源页面上

5.2.3 价格

设置每晚房价和可选费用。价格以人民币为单位，向房客显示透明明细。

每晚基础价格（必填）：费用前的每晚房价，输入人民币金额（如200），创建后可按季节调整

清洁费（可选）：一次性费用（非每晚），建议¥50-150，在房客价格明细中单独显示

房客支付： 基础价格×晚数 + 清洁费 + 税费 | 您收到： 基础价格×晚数 + 清洁费 - 平台费（15%-等级%）

5.2.4 照片

上传房源的高质量图片。照片是预订决策的首要因素。

添加房源照片

  • 点击”添加更多”从设备选择照片
  • 每张照片上传时显示”新”徽章
  • 最少：5张照片（建议：10-20张）
  • 最大文件大小：每张10MB
  • 支持格式：JPG、PNG

照片提示： 使用自然光、展示所有房间（浴室/厨房/卧室）、从角落拍照展示全景、第一张是封面要精心选择

重新排序： 拖动重新排序，第一张显示在搜索结果，按顺序讲述故事（入口→起居区→卧室→户外）

添加视频（可选）： 粘贴YouTube URL，为房客提供虚拟导览，每个房源最多1个视频

5.2.5 可用性

在创建房源时设置屏蔽日期。高级日历功能（自定义定价、周期规则）在创建房源后可用。

屏蔽日期： 选择房源不可用的日期（注意：完整日历功能将在创建房源后可用）

房源创建后： 完整日历、屏蔽日期范围、自定义定价（节假日/旺季）、最短/最长入住、提前预订窗口、周期可用性规则

5.2.6 设施与无障碍

选择您房源提供的所有设施和无障碍功能。准确选择可提高搜索匹配度，帮助房客找到满足其需求的房源。

您提供什么设施？

已选设施显示粉色背景和红色勾号：无线网络、厨房、洗衣机、烘干机、电视、阳台、花园、停车位

未选设施显示灰色背景：空调、暖气、泳池、热水浴缸、健身房、壁炉、洗碗机、洗衣、熨斗

无障碍功能： 无台阶入口、宽门、无障碍浴室、扶手、电梯、轮椅无障碍、听力无障碍、视觉辅助、允许服务动物

注意： 设施和无障碍选项从数据库动态加载。

5.2.7 预订规则

配置取消政策、即时预订设置和最短入住要求。

最短入住晚数： 设置最短预订长度（默认3晚，范围1-30晚），创建后可按日期范围调整

取消政策： | 政策 | 全额退款期限 | 部分退款 | 无退款 | |—–|————|———|——-| | 灵活 | 入住前24h+ | - | <24h | | 适中 | 入住前5天+ | 2-5天(50%) | <2天 | | 严格 | 入住前7天+ | 7-30天(50%) | <7天 |

房东取消后果： 客人全额退款、可能被收取费用、影响搜索排名

重要退款规则： 清洁费和税费（入住前取消始终100%退款）、基础价格（根据政策百分比）、服务费（灵活/中等可退，严格不可退）

示例（中等政策，入住前3天取消）： 基础¥200→50%=¥100、清洁¥50→100%、税费¥20→100%、服务费¥28→100% = 总退款¥198

预订偏好

即时预订（开关）：启用后房客可直接预订无需批准，支付自动授权。条件：所有人（不推荐）、已验证房客、回头客、好评房客（>4.5）

自助入住（开关）：启用后房客可自行入住，需提供详细入住说明（密码箱/钥匙/智能锁），说明在到达前24小时自动发送

5.2.8 房屋规则

设置入住/退房时间和房源规则以设定房客期望。

入住/退房时间： 默认入住14:00、退房11:00，点击可更改，在预订确认中向房客显示

房屋规则

切换每条规则开/关：允许吸烟（关闭=禁止室内吸烟）、允许宠物（关闭=禁止宠物）、允许派对/活动（关闭=禁止聚会）- 默认：全部关闭（禁止）

安静时间（开关）：启用以强制安静时段 - 开始时间（默认：晚上11:00）、结束时间（默认：早上8:00）- 显示在房源页面的房屋规则中

5.2.9 预览与提交

预览： 点击”预览”（右上角）在提交前查看房客视图

保存行为：

  • 创建新房源： 更改仅自动保存到您的设备（不在服务器上）。准备好后点击”提交审核”发送给审核员。
  • 编辑活跃房源： 更改自动分类：
    • 关键更改（卧室、浴室、容量、房源类型、主要设施）：保存但在审核员批准前对房客不可见（24小时审核）
    • 自动批准更改（照片、标题、描述、价格、WiFi密码、入住说明、日历）：立即生效，无需审核

详见5.4.3节了解两级审核系统的完整详情。

5.3 编辑房源

↑ 返回目录

房源标签 → 点击房源 → 三点菜单（⋯）→ “编辑房源”。更改自动分类：关键（卧室、浴室等）需审核员批准，自动批准（照片、价格等）立即生效。

智能定价折扣： | 类型 | 适用条件 | 推荐折扣 | |—–|———|———| | 周折扣 | 7-27晚 | 10-20% | | 月折扣 | 28晚以上 | 20-30% | | 即时优惠 | 当日预订（白金/钻石客人） | 30-40% |

规则： 仅适用单个最优折扣，不叠加。

5.3.2 编辑可用性

屏蔽日期： 点击日期 → “屏蔽”按钮 | 取消屏蔽： 点击已屏蔽日期 → “取消屏蔽”

操作按钮： 导入日历（Airbnb/VRBO同步）、周期规则（自动屏蔽）、导出日历

5.3.3 编辑照片与视频

照片： 点击X删除，拖动重排序（第一张=封面），“添加更多”上传新照片

视频： 粘贴YouTube URL → 显示在轮播首位，可提高30%+预订转化率

5.3.4 管理房型（仅限酒店/青旅）

功能概述： 酒店、青旅可以定义多个房型或床型，每个都有自己的定价、容量、设施和可用性。

术语：

  • 酒店 → “房型”（如标准房、豪华套房）
  • 青旅 → “床型”（如4床混合宿舍上铺、女生宿舍单床）

左图：酒店房型列表，显示豪华大床房(¥200/晚)、豪华双床房(¥250/晚)、普通双床房(¥180/晚)；右图：青旅床型列表，显示四人间(¥60/晚)、六人间(¥50/晚)

界面说明：

  • 取消 - 左上角，点击返回房源编辑页面
  • + - 右上角，点击添加新的房型/床型
  • 房型/床型卡片 - 显示名称、每晚价格、床位数量、最大入住人数
  • 管理房间 - 点击查看和管理该类型下的具体房间

⚠️ 发布要求： 多单元房源类型（酒店、青旅）在至少有一个活跃房型/床型之前不能发布。如果您尝试在没有房型的情况下提交，会看到验证错误。

为什么房型/床型重要：

与房客预订整个房源的度假租赁不同，酒店和青旅：提供多种单元类型、允许同一日期多个预订、按房型/床型追踪容量、允许不同住宿有不同定价

访问房型/床型管理： 房源标签 → 点击酒店/青旅房源 → 滚动到”房型”或”床型”部分 → 点击”管理类型”

A. 创建房型/床型：

左图：添加酒店房型表单；右图：添加青旅床型表单

表单字段说明：

区域 字段 说明
基本信息 房型名称 * 必填，如”豪华大床房”、“6人混住宿舍”
单元类型代码 可选，内部参考代码（如DLX-K）
描述 可选，最多500字符
容量 床位数量 该房型/床型的床位数
床型 如”特大床”、“大床”、“上下铺”
每单元最大入住人数 每个房间/床位最多可住几人
最少入住人数 最少需要几人才能预订

创建步骤： 1. 在房型/床型列表页点击右上角 + 按钮 2. 填写房型/床型名称（必填） 3. 设置床位数量和入住人数 4. 继续滚动设置价格、库存等信息 5. 点击保存

B. 编辑房型/床型： 点击卡片 → 修改字段 → 保存。可编辑：名称、总单元数、容量、价格、描述、设施、照片。注意： 有活跃预订时不能减少单元数低于预订数。

C. 可用性和容量：

容量如何运作： 每个房型/床型有自己的容量（总单元数）→ 多位房客可同一日期预订相同类型 → 系统自动追踪预订vs总数 → 数据库级别防止超额预订

示例：

豪华大床房：总共5个单元

- 预订1：房客A预订1间（12月1-3日）→ 剩余4间
- 预订2：房客B预订2间（12月1-3日）→ 剩余2间
- 预订3：房客C尝试预订3间（12月1-3日）→ 被阻止（仅2间可用）

查看可用性： 每个房型/床型显示当前可用性 - 绿色（可用）、黄色（有限）、红色（已满）- 点击”查看日历”查看详情

D. 停用/删除房型/床型：

停用（临时）： 点击房型 → 关闭”活跃”开关 → 不再接受新预订（现有预订有效）→ 可随时重新激活

删除（永久）： 点击房型 → 点击”删除” → 确认。注意： 有活跃预订则不能删除

E. 酒店/青旅最佳实践： 使用清晰名称（“带阳台的豪华大床房”比”A房间”好）、添加房型特定照片、战略性定价、定期监控可用性、保持描述准确更新

常见房型示例（酒店）： 标准双床房、豪华大床房、带客厅的套房、家庭房、无障碍房

常见床型示例（青旅）： 4床混合宿舍上铺、6床女生宿舍上铺、8床混合宿舍单床、私人单人间、私人双人间

组合设施 = 共享设施 ∪ 房型设施

示例：

  • 您酒店的共享设施： WiFi、泳池、健身房、餐厅
  • 豪华房设施：阳台、迷你吧
  • 标准房设施：（无）

5.4 房源审核与批准流程 [仅限房东]

↑ 返回目录

了解房源状态和编辑如何被审核有助于您有效管理房源。

5.4.1 房源状态

您房源的状态决定房客是否可以看到和预订。

状态 含义 房客可见？ 可预订？ 您的操作
待审核 新房源已提交，等待审核员批准 ❌ 否 ❌ 否 等待审核员审核（24-48小时）
请求修改 审核员退回要求修改 ❌ 否 ❌ 否 进行请求的修改，然后立即重新提交
活跃 已批准并在平台上线 ✅ 是 ✅ 是 管理预订，可随时暂停
活跃（待审核编辑） 活跃房源有关键编辑等待审核员批准 ✅ 是（显示旧批准数据） ✅ 是 等待审核员审核您的更改
您暂停的 您临时禁用了它 ❌ 否 ❌ 否 准备好时取消暂停（无需重新审核）
审核员暂停的 审核员发现上线房源问题 ❌ 否 ❌ 否 修复问题，然后重新提交审核
已拒绝 审核员永久封锁此房源 ❌ 否 ❌ 否 修复问题并立即重新提交，或通过聊天申诉
已归档 您永久移除了它 ❌ 否 ❌ 否 无法恢复

重要说明：

无草稿状态： 创建中的房源仅保存在设备上，提交后创建状态为”待审核”。

有待审核编辑的活跃房源： 编辑关键信息时房源保持活跃可预订，房客看到旧的批准信息直到新更改被批准。

5.4.2 新房源提交工作流程

创建您的第一个房源：

  1. 开始创建： 在编辑器中开始填写房源详情（进度仅自动保存到设备，可退出继续）

  2. 提交审核： 完成后点击”提交审核”→ 房源状态变为待审核→ 您将收到通知

  3. 审核员审核（24-48小时）：

    选项A：批准 → 状态变为活跃，房源上线可预订

    选项B：请求修改 → 状态变为请求修改，您收到具体反馈（照片、描述、设施、定价等）→ 修改后重新提交

    选项C：拒绝 → 状态变为已拒绝（欺诈、违规、重复等）→ 修复后重新提交或申诉

5.4.3 编辑活跃房源：两级审核系统

一旦您的房源活跃，编辑分为两类。这平衡了您的自主权和质量控制。

🔴 关键更改（需要审核员重新批准）

这些从根本上定义房客预订的内容。更改需要审核员批准后才能上线。

示例：

  • 房源类型（独栋房屋 → 公寓）
  • 卧室数量
  • 浴室数量
  • 最大房客容量
  • 位置/地址
  • 主要设施（泳池、热水浴缸、停车、空调/暖气、厨房、洗衣机/烘干机、无障碍功能）

发生什么： 保存后房源保持活跃可预订 → 房客看旧批准信息 → 审核员24小时内审核 → 批准后更改上线 / 拒绝则保持旧信息

重要： 待审核期间的预订基于旧批准信息，您必须履行。

✅ 自动批准更改（立即上线）

所有其他不从根本上改变房客预订内容的更改。更新立即上线，无需审核员审核。

示例： 照片/视频、标题、描述、价格、清洁费、取消政策、最短/最长入住、次要设施、WiFi密码、门禁码/入住说明、本地推荐、社区指南、停车说明、垃圾/回收信息、日历可用性

发生什么： 您保存更改 → 立即上线 → 房源保持活跃 → 房客立即看到新信息 → 无需审核

5.4.4 管理活跃房源

批准后，您有完全控制权：

自行暂停： 点击”暂停房源”临时禁用，随时取消暂停无需重新审核

审核员暂停： 审核员发现问题后暂停，修复后重新提交，24小时内审核

归档： 永久移除，无法恢复，预订历史保留用于税务记录

5.4.5 评价审核员

房源获批后，您可以对审核员的服务进行评价。

评价入口： 收到审批通知后 → 点击”评价审核员”

评价项目： | 项目 | 说明 | |—–|——| | 综合评分 | 1-5星整体满意度 | | 帮助度 | 反馈是否有建设性 | | 响应速度 | 审核是否及时 | | 专业度 | 沟通是否专业 | | 文字反馈 | 可选，具体意见 |
